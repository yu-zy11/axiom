# AxiomKernel 用户可读错误文案映射表

本文档定义 `AxiomKernel` 内部错误码到用户可读文案的映射规则，目标是为上层 CAD、脚本平台、自动化工具和 Web/桌面 UI 提供统一、清晰、可本地化的错误提示文本。

## 1. 文档目标

本文档用于明确：

- 哪些错误码需要用户可读映射
- 用户看到的文案应如何组织
- 技术错误与用户提示如何分层
- 建议的操作指引模板

## 2. 设计原则

用户可读错误文案必须满足以下原则：

- 尽量解释“发生了什么”
- 尽量提示“可以怎么做”
- 不把底层算法实现细节直接暴露给普通用户
- 同一错误码在同一产品内的表达保持稳定
- 支持后续多语言本地化

## 3. 文案结构建议

每条映射建议至少包含：

- `错误码`
- `短文案`
- `详细说明`
- `建议动作`
- `是否适合直接展示给终端用户`

推荐数据结构：

```json
{
  "code": "AXM-BOOL-E-0005",
  "title": "布尔运算失败",
  "message": "模型在局部区域无法稳定分类，当前运算未能完成。",
  "suggestion": "请尝试简化接触区域、减小局部复杂度，或启用修复模式后重试。",
  "userVisible": true
}
```

## 4. 展示分层建议

### 4.1 面向终端用户

建议展示：

- 短标题
- 简短说明
- 一条可执行建议

避免展示：

- 复杂数值术语
- 内部模块名
- 多层嵌套错误栈

### 4.2 面向高级用户或工程师

可以额外展示：

- 主错误码
- 诊断 ID
- 关联实体
- 详细技术说明

|---|---|---|---|
| `AXM-CORE-E-0001` | 输入对象无效 | 当前操作引用的对象不存在或已失效。 | 请检查输入对象是否已正确创建或仍在当前版本中可用。 |
| `AXM-CORE-E-0002` | 参数超出范围 | 输入参数超过当前操作允许范围。 | 请检查尺寸、角度、距离或索引参数是否合理。 |
| `AXM-CORE-E-0004` | 当前操作不受支持 | 当前输入或模式不支持此操作。 | 请更换输入类型或使用兼容模式。 |
| `AXM-CORE-E-0008` | 版本不兼容 | 当前数据与引擎或插件版本不匹配。 | 请升级兼容组件，或重新导出标准格式后重试。 |

## 6. 警告文案映射

## 7. UI 展示建议

### 7.1 弹窗型错误

适合展示：

- 标题
- 一句话错误说明
- 一条操作建议

示例：

- 标题：`布尔运算失败`
- 正文：`模型在局部区域无法稳定分类，当前操作未完成。`
- 建议：`请尝试简化接触区域，或启用修复模式后重试。`

### 7.2 面板型错误

适合展示：

- 标题
- 主文案
- 详细问题列表
- 关联对象
- 错误码
- 诊断 ID

### 7.3 日志型错误

适合展示：

- 时间
- 错误码
- 简短文案
- 诊断 ID

## 8. 国际化建议

建议内部存储以下字段：

- `title_zh_cn`
- `message_zh_cn`
- `suggestion_zh_cn`
- `title_en_us`
- `message_en_us`
- `suggestion_en_us`

注意：

- 错误码本身不做本地化
- 诊断 ID 不做本地化
- 建议动作应避免语言强绑定的技术缩写

## 9. 测试要求

用户可读文案映射表也需要测试。

必测项：

- 每个 `P0` 错误码都存在用户文案映射
- 同一错误码在 UI 层文案一致
- 缺失映射时存在默认回退文案

默认回退文案建议：

- 标题：`操作失败`
- 正文：`当前操作未能完成，请查看详细诊断信息。`
- 建议：`请联系开发人员并提供错误码与诊断 ID。`

## 10. 结论

这份映射表的作用是把底层错误体系转化为上层用户真正能理解的提示层。只要错误码稳定、文案映射清晰，上层产品就能在不暴露复杂实现细节的前提下，给用户足够有帮助的反馈。
